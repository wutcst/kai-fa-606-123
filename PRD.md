# Starline Valkyrie 产品需求文档 PRD

## 1. 产品概述

`Starline Valkyrie` 是一款面向课程实训展示的横版太空射击游戏。玩家驾驶战机在横向星际战场中自动开火，通过移动走位、拾取道具、释放 BOMB、击败多阶段 Boss 和积累连击获得更高分数。

产品定位不是复杂硬核飞行模拟，而是“够爽、够无脑、打开就能玩”的街机射击体验。

## 2. 背景与问题

课程实训要求完成一个可展示、可运行、可提交源码和报告的项目。原始 starter 工程偏文本交互，不利于直观展示游戏完成度。因此项目选择网页 Canvas 游戏方向，用图形化、音效、特效、Boss 和打包发布提升展示效果。

目标问题：

- 需要一个能在答辩时直接演示的完整游戏。
- 需要玩法足够直接，不需要长时间教学。
- 需要在浏览器和 Windows 上运行，便于小组成员和老师体验。
- 需要有测试、README、报告和 release 包支撑课程提交。

## 3. 目标用户

| 用户 | 需求 |
| --- | --- |
| 玩家 / 体验者 | 快速开始游戏，感受射击、爆炸、Boss 和音效反馈。 |
| 课程验收老师 | 能看到完整功能、工程结构、测试、文档和运行成果。 |
| 小组开发成员 | 能理解代码结构，继续调参、修 bug、准备答辩材料。 |

## 4. 产品目标

### 4.1 核心目标

- 玩家打开游戏后 5 秒内理解操作并开始游玩。
- 游戏具备完整闭环：开始、战斗、成长、Boss、死亡、记录、重开。
- 画面和音效具备强烈打击感，适合短时间答辩展示。
- 支持网页运行和 Windows release 包体验。

### 4.2 非目标

- 不做联网排行榜。
- 不做账号登录。
- 不做复杂剧情系统。
- 不做多人联机。
- 不做真实素材版权依赖，当前视觉资源以程序化绘制为主。

## 5. 核心玩法

玩家控制战机在屏幕内移动，系统自动向前射击并轻微自动索敌。敌人从右侧进入，玩家需要躲避敌弹和敌机撞击，击杀敌人获取分数、连击和掉落。玩家可通过道具提升火力、回血、充能 BOMB、切换激光或导弹攻击模式。每 30 秒左右出现一层 Boss，Boss 难度随时间提升。

## 7. 非功能需求

| 类型 | 需求 |
| --- | --- |
| 性能 | 普通电脑上应保持流畅，粒子、敌弹、掉落和冲击波数量必须有上限。 |

| 安全性 | Electron 禁用 Node 集成，开启上下文隔离和沙箱。 |

## 8. 用户流程

### 8.1 首次游玩

1. 用户打开网页或 Windows exe。
2. 默认进入 Normal 难度。
3. 用户通过键盘或触控移动战机。
4. 系统自动开火。
5. 用户击杀敌人、拾取道具、释放 BOMB。
6. Boss 出现并进入阶段战斗。
7. 玩家死亡后显示 Game Over 和成绩。
8. 游戏保存本次记录，用户可重开。

### 8.2 切换难度

1. 用户点击 HUD 中 Easy、Normal 或 Hard。
2. 系统保存难度选择。
3. 当前局重新开始。
4. HUD 显示该难度下的最佳成绩。

## 12. 风险与应对

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| 粒子过多导致卡顿 | 游戏体验下降 | 设置实体数量上限，缓存背景，限制 Canvas 像素比。 |
| 特效遮挡敌弹 | 玩家误伤，体验差 | 粒子聚焦于命中点，敌弹后绘制并保持高对比颜色。 |
| Windows 本地访问失败 | 无法体验网页版 | 提供 Electron release 包作为替代。 |
| Electron 包体较大 | 上传和下载较慢 | 以 zip 形式交付，说明需完整解压运行。 |
| 未签名 exe 被系统提示 | 用户误以为无法运行 | 在发布说明中说明未签名原因，后续可补代码签名。 |

## 13. 后续需求池

- 设置页：音量、画质、震屏强度、粒子强度。
- 更多道具：护盾、时间减速、自动吸附、双倍分数。
- 更多敌人：激光炮台、护盾敌机、自爆敌机。
- 更多 Boss：分身、弱点暴露、阶段转场。
- 成就系统：首次 Boss 击杀、无伤 60 秒、连击达标。
- 本地排行榜：按难度保存多条历史成绩。
